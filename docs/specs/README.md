## Spec Template

When creating a new spec, include:
1. **Problem** — what user need does this address?
2. **Solution** — high-level approach
3. **Design** — detailed design with code examples if applicable
4. **Testing** — how to verify the implementation
5. **Open Questions** — unresolved decisions
